chapter 1. 对象的基本概念

这一章还是涉及到不少基础语法相关的东西的,这些关于基础语法的东西我就不再记录了.

首先,c也是可以采用一些面向对象的手法.比如说使用struct将数据打包,然后定义一些函数作用于这些struct.而c++使用class的话,就直接将函数和数据封装在一起了.

那么封装所带来的额外成本是什么?

其实c++对象在布局和存取时间上的额外负担是virtual造成:

  • virtual function,支持动态绑定.
  • virtual bass class,对于多次出现在继承体系的base class,有被共享的一个base class实体.

1.1 The C++ Object Model

数据成员分为:static,nonstatic;函数成员分为:static,nonstatic,virtual.

关于对象模型的表示,大致有以下几种.

第三个大致是实际的c++所使用的模式.

其中所有static属性的成员都处于此对象内存之外,非虚函数也是.

class object之内,除了nonstatic data member,还有一个虚函数指针(vptr),指向一个虚函数表(vtbl),这个表头含有 type_info object.其中constructor,destructor和copy assignment会做一些有关于vptr的动作.

1.2 A Keyword Distinction

首先来讨论一下,在c++中使用struct和class的区别:

其实,c++之所以要引入struct,很大程度上是为了兼容现存的c代码.但如果这样,有必要引入class吗?其实引入class不单单是引入关键词这么简单,还有它所支持的继承和封装的哲学.

甚至可以主张说.它的作用在于使c程序员更方便的入手c++.

1.3 An Object Distinction

C++支持三种程序设计典范:

  • Procedural model(面向过程).
  • Abstract data type model(ADT).数据和其操作封装到一起.
  • Object- oriented model(面向对象模型).除了数据和操作封装意外,还支持继承,多态等.

**多态的实现是借助基类reference或者pointer的.**直接使用类对象的话,会造成slicing.

ADT模式和OO模式对象的显著区别是什么?

ADT的对象的类型是固定的且确定的,在编译时期就定义好了.

而OO模式中,只能根据继承体系大致界定一个对象的类型(派生于某个基类),但是具体是哪一个不确定,只有在执行时才可以确定(动态绑定).

C++支持多态的方法:

  • 隐含的转化操作,将 derived class指针转化为指向 public base type的指针.
  • 借助 virtual function机制.
  • 借助 dynamic_cast和 typeid运算符.

多态主要用于为不同的类通过一个通用的接口来进行封装,这个通用接口被定义在base class中.

需要多少内存才能表示一个class object?

  • nonstatic data members的大小.
  • 由于对齐机制填补的空间.
  • 由于支持virtual额外的空间,比如说vptr指针.

不同类型的指针的区别是什么呢?“指针类型”的意义在于决定编译器接受内存内容的方式,而cast不影响内存内容,只影响编译器解释内存内容的方式.

对于指向Bear对象,但是分别为Bear和ZooAnimal类型的指针,所指向的地址相同,但是其解释的内存内容所包含的范围不同.一个是整个Bear对象,一个只含有ZooAnimal部分.

当Slicing发生时,确实将ZooAnimal部分的内存进行了拷贝,但为什么vptr不能支持派生类的多态呢?

chapter 2. 构造函数语义

2.1 Default Constructor Construction

C++ ARM中有一句非常含糊的话:“当编译器需要时,会生成默认的构造函数”.“编译器需要”和“程序员需要”是两个不同的概念,所以也并不是不声明就一定生成构造函数,即使生成了,也要注意往往不能达到开发者的预期.

首先讨论的是所谓的c++编译器所需要的场景.

1)如果有一个含有default构造函数的Member Object

首先需要注意的是,默认构造函数指的是:没有参数或者参数有默认值的.

这一般发生于member Object含有默认构造函数的情况.首先如下所示:

class Foo { 
    public: 
        Foo(); Foo(int);
    private:
        int num_;
};
class Bar { 
    public: 
        //Bar() { str = NULL; }
        Foo foo; 
        char *str; 
};
int main(){  
 	Bar bar;//确实生成默认的构造函数,并且其中的构造函数中调用了foo   
}

此时会生成一个默认的Bar的构造函数,相当于Bar bar这行代码处,隐式地调用了生成的Bar构造函数,并且在生成的构造函数中还调用了Foo的默认构造函数.

如果生明并定义了构造函数,就可以避免编译器的生成行为了吗?答案是不可以的.考虑定一个Bar构造函数:Bar() { str = NULL; }.但实际上编译器的行为如下:

Bar() { str = NULL; } 
//实际呈现的行为
inline Bar()
  : foo{Foo()}  {
   this->str = static_cast<char *>(NULL);
 }

我们发现新插入了一个借助Foo构造函数初始化foo的现象.如果出现多个Object,那么也是这样的道理,指的是构造函数没有声明或者所定义的构造函数中缺少对某些项的初始化.如图中所示:

inline Snow_White()
: dopey{Dopey()}
, sneezy{Sneezy(1024)}
, bashful{Bashful()}
{
  this->mumble = 2048;
}

总的来说,对于含有Member Object而且其还有默认构造函数的情况,编译会为其生成default构造函数,如果我们已经定义了,如果没有对这些Member Obeject 进行初始化,那么编译器会对构造函数插入新的代码(调用成员对象的默认构造函数进行初始化).

2)基类有默认构造函数

这种情况和上面的情况是同理的.如果对于某个派生类已经定义了多个构造函数,但是就是没有默认构造函数,而其基类有默认构造函数的情况下,这个时候此派生类会将调用基类默认构造函数的代码插入到已定义的构造函数中,但这种情况下,该派生类就不再会生成默认构造函数了.

class Foo { 
    public: 
        //Foo(); 
        Foo(int num) { num_ = 1; }//如果将这里注释掉,编译器则可以生成Foo默认构造函数
    private:
        int num_;
};
class Bar: public Foo { 
    public: 
  		Bar(int num) { str = NULL; }
      char *str; 
};

对于图中的这种情况,这个时候编译器需要调用Foo的默认构造函数,但是Foo已经定义了其他非默认构造函数,这个时候就不会再生成Foo的默认构造函数,因此最终会报错.

error: constructor for 'Bar' must explicitly initialize the base class 'Foo' which does not have a default constructor

含有虚函数的类

“含有”指的是,要么这个类本身定义了虚函数,或者继承了一个虚函数,或者其继承体系中,含有虚函数.

在这时会有扩张操作发生:

  • 一个 virtual function table被产生.
  • 每一个 class object中,一个额外的指针成员会被合成,内有 class vtbl.

对于一个类中的 class objects,编译器会合成每一个类对象中的vptr.

Class with a Virtual Base Class

对于含有一个 virtual base class的派生类,编译器必须使其在运行时明确基类的地址.

????

最后梳理一下,总共有四种情况,导致需要编译器需要合成 default constructor.

在合成的 defuault constructor中,只有 base class subobjects和 member class objects被初始化.

所以注意两个误解:

  • 如果没有定义 default constructor就会合成.
  • 合成的 default constructor会设定每一个数据成员.

2.2 拷贝构造函数

首先说明一下,拷贝构造发生的时机:X xx = x;对象传参,返回对象.

Default Memberwize

当没有提供 explicit copy constructor时,如果要发生拷贝构造,内部是以“逐个成员进行拷贝”的手法.对于其中的 member class object,则以递归的方式进行“逐个成员拷贝”.

比如说

class Word{
  public://......
  private:
  	int occur_;
    String word_;
};

上面的这种情况,default memberwise initialzation会拷贝 occur_,然后递归地处理 word_.

但是生成拷贝构造函数的时机又是什么时候呢?不声明explicit copy constructor就一定会合成吗?这与该类是否呈现Bitwise Copy Semantics有关.

chapter 3. The Semantics of Data

首先考虑一个菱形继承的例子(X,Y,Z,A).调用 sizeof打印出来的结果依次为:1,8,8,12.

对于X,这个空的类,其大小为1,其实是编译器预留出来的一个 char,其作用在于使不同的类在内存中地址不同.

对于Z,Y虚拟继承了X,其内存占用主要来自于:

  • 支持继承带来的负担.通常是一个指针或者偏移量.
  • 编译器对于特殊情况的优化处理.
  • Alignment,对齐机制.比如说32位机器上,会对齐到8个字节.

对于有12个字节的A应该是这这种情况:

  • 确保唯一性的 char,1B.
  • class A自身,0 B.
  • Base class X,Y,8 B
  • alignment.

3.1 The Binding of a Data Member

在一个类中定义了x,外部也有一个x全局变量.如果类的成员函数引用这个x,指的究竟是哪一个呢?就现在而言,是内部的x.

如果一个内联函数在一个class声明后被定义,就需要重新评估求值.

比如说下面的这一段:

extern int x;
class Point3d{
public:
  	float X() const { return x; }//这就是一个inline函数,对于该函数的分析,会等到整个类声明出现了为止.
private://像这种再类内部定义的函数位inline函数,会等到整个class都声明完后进行
  	float x;
};

但是参数列表中就不是这样的……

3.2 Data Member Layout

这一届主要探讨的是member在类中的排列顺序问题.

Nonstatic data members在对象中的排列顺序和其被声明的顺序一样.中间插入的 static data members不会计入.因为有时候有 alignment填空,更准确地说,同一个access,不一定连续较晚出现的成员的地址较高.

除此之外,在含有 virtual function class的对象中还以有一个 vptr.

目前的各家编译器,一般是将 access section连锁在一起,按照声明顺序形成连续的区块.

3.3 Access of a Data Member

我们将讨论 nonstatic和 static两种情况.

Static Data Member

最突出的特点就是,其地址在class之外,其实体是唯一的,处于数据段中.

当访问时的编译器的转化如下:

//origin.chunksize = 250;
Point3d::chunkSize = 250;

尽管看上去是经过一个对象的“.”或者“->”实现,但实际上对于 static类型来说,并不需要经过 class object.

并且这种特征不受继承特征的影响.

Nontatic Data Members

直接处于 class object中,除了借助 explot和 implicit的class object,否则无法存取.

**当在成员函数处理 nonstatic members,就会有 implicit class object生成.**其实都是借助了一个 implicit class object,this指针.
